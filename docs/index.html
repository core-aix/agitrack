9d04e17 why this exists

Agent code without provenance is unreviewable.

A coding agent edits twelve files while you get coffee; a week later nobody can say which prompt produced which change, or what it cost. aGiTrack makes the unit of agent work a commit, so review, blame, bisect, and revert work on agent output just like yours.

a11m0de pick your workflow

Four ways to run it, your call.

Two independent choices, how you run it (TUI or background) and when commits happen (each turn, or on your own git commit), combine into four modes. All record the same per-turn trace and tokens; switch between them freely.

Interactive · Auto default

Watch the agent's real TUI; aGiTrack commits every finished turn and merges it from an isolated worktree. Hands-off, just work, get a clean traced history.

Interactive · Manual

Feels like plain git: you decide when to commit. Pending turns fold into your own commit, one tidy commit with your code and the full AI trace.

Background · Auto

Prefer a GUI? Drive the agent from any UI, the Claude desktop app, an IDE extension, a chat window, while aGiTrack tracks it headlessly and commits each turn. Runs as a daemon; agitrack -b stop when done.

Background · Manual

Track a session you drive yourself, commit on your own terms. The AI trace folds into your commit, and aGiTrack can even start itself on git commit so nothing goes untracked after a reboot.

Only the interactive-auto default uses an isolated worktree; the other three edit your current branch directly. A commit with no AI work is never touched.

b7e1d09 <aGiTrack> see the whole repo at a glance

A live dashboard of what the agents did.

Run agitrack -d for a live, auto-refreshing dashboard on localhost. Every number comes from commit metadata alone, identical on every clone, no database, no tracking server.

agitrack -d · localhost
agitrack -d
starts a background daemon on localhost and opens your browser, then returns to your shell; it stops when the terminal closes or with agitrack -d stop
a1f8b28 <aGiTrack> reconstruct history from past sessions

Didn't use it from day one? Show the history anyway.

Already built your project with Claude or OpenCode, without aGiTrack? Run agitrack --backtrace. It reads the coding-agent transcripts already on your machine and reconstructs what those sessions did here: tokens, models, lines changed, the file-by-file history, and the full conversation behind every change, in the same dashboard, clearly marked as a historical view. No prior aGiTrack use, nothing uploaded, and it works even in a folder that was never a git repo.

Want it in your real git history? agitrack --backtrace commit replays your commits onto a new branch and stamps the aGiTrack metadata onto the ones your agents produced, keeping your own commits verbatim. It only ever writes to a new branch and leaves your current one untouched; just run it and follow the prompts.

agitrack --backtrace
reconstructs and serves the dashboard from local transcripts (a background daemon; agitrack --backtrace stop ends it)
agitrack --backtrace commit
bakes the reconstruction into real commits on a new branch, then follow the printed steps (needs a clean working tree; your current branch is left alone)
